Running through 5 security layers…
Layer 1 — Input sanitisationActive

First gate. Every incoming request is scanned before any ML model sees it.

📌 Regex pattern scanner — 15+ known injection/zero-day patterns
📌 Perplexity filter — Character entropy proxy. High entropy = obfuscated text
📌 Input normalisation — Unicode normalisation, whitespace collapsing
Threshold: perplexity > 150 → FLAG
Pattern match → immediate BLOCK escalation
Layer 2 — Embedding anomalyActive

ML-based semantic analysis. Understands intent, not just surface patterns.

🤖 DeBERTa classifier — microsoft/deberta-v3-base fine-tuned on injection dataset
🤖 Variational Autoencoder — Reconstruction error = anomaly score for zero-days
🤖 Keyword feature extractor — 30+ weighted security-relevant tokens
DeBERTa > 0.50 → injection FLAG
VAE error > 2.5 → zero-day FLAG
Layer 3 — Ensemble engineTraining

Fuses all signals into a single threat score. Core of the framework.

⚡ GNN session graph — Models API call sequences as a graph. Catches behavioural anomalies
⚡ XGBoost meta-learner — 8-feature ensemble. Final score = weighted prediction
⚡ MITRE ATT&CK mapping — Every threat tagged to MITRE technique ID
Ensemble score > 0.75 → BLOCK
Score 0.45–0.75 → FLAG · <0.45 → PASS
Layer 4 — Policy enforcementActive

Dual-LLM verifier architecture reduces false positives before enforcement.

🛡 Primary LLM — Renders verdict: BLOCK / FLAG / PASS
🛡 Verifier LLM — Independent second opinion. Can override to reduce false positives
🛡 Policy engine — Applies action: terminate, quarantine, or forward
BLOCK → terminate + log + alert
FLAG → quarantine + manual review queue
Layer 5 — Output validation & auditActive
Output sanitisation

LLM responses are scanned for PII leakage, prompt echoing, and policy violations before delivery.

Audit logging

Every request — input hash, all layer scores, verdict, latency — written to ELK stack with 90-day retention.

Feedback loop

Human-reviewed false positives/negatives fed back to retrain DeBERTa and update XGBoost features.

Project overview

Multi-Layer AI Security Framework for Detecting Zero-Day Vulnerabilities and Prompt Injection Attacks

Final year project, Department of Computer Science & Engineering, Dayananda Sagar Academy of Technology and Management (VTU).

Guide: Dr. ATI NARAYANA S | Team: 4 members

Architecture summary

A five-layer defense-in-depth pipeline: every request passes through input sanitisation → embedding anomaly detection → ensemble engine → policy enforcement → output validation. No single layer is a single point of failure.

ML components

DeBERTa-v3-base Variational Autoencoder Graph Neural Network XGBoost meta-learner Dual-LLM verifier Perplexity filter Token gradient analysis

Tech stack
